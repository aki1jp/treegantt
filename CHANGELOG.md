# 変更履歴 (Changelog)

本ファイルは TreeGantt の**製品リリース履歴**を記録します。
形式は [Keep a Changelog](https://keepachangelog.com/ja/1.1.0/)、バージョニングは [セマンティックバージョニング](https://semver.org/lang/ja/) に従います。

設計の変更点・要点は設計書 `docs/treegantt_design.md` の「改訂履歴」に記録します（役割分担: 製品リリース=本ファイル / 設計=設計書）。

## [1.9.0] - 2026-07-11

### 追加
- **多言語対応（第2弾）**: 英語表示の対象範囲を拡大。右クリックのコンテキストメニュー・競合解決ダイアログ・タスク削除確認・エラー画面・リソースビュー・プロジェクトタブの右クリックメニュー・クロスプロジェクト参照の管理画面・各種操作失敗時のトースト通知を英語表示に対応。WBSグリッドのステータス/優先度表示、タスク複製時のタイトル末尾（「(コピー)」/「(Copy)」）、今日ラインの時刻表示もあわせて切り替わる。これによりフロントエンドの静的な画面文言はおおむね日本語/英語の切り替えに対応した。

## [1.8.0] - 2026-07-10

### 追加
- **多言語対応（日本語/英語切り替え、第1弾）**: 画面右上に言語切替ボタン（「日本語」/「English」）を追加。ツールバー・アプリ本体（確認ダイアログ・トースト通知を含む）・タスク/マイルストーンの編集モーダル・WBS/ガント本体（列見出し・展開/折りたたみ・行追加欄）を英語表示に切り替えられる。設定は端末に保存され次回起動時も保持される。タスクタイトルや担当者名などユーザーが入力した内容は翻訳の対象外。コンテキストメニュー・リソースビュー・プロジェクト管理画面など一部の画面は今後の対応となる。

## [1.7.0] - 2026-07-08

### 追加
- **タスクの色によるフィルタ**: ツールバーのフィルタ列に「色」セレクタを追加。「すべて」「色付き」または使用中の色ごと（丸印スウォッチ表示）で絞り込みできる。色の同一性は背景色を優先し、背景色が未設定のタスクは文字色で判定する。

## [1.6.0] - 2026-07-08

### 追加
- **クロスプロジェクトのタスク参照**: 他プロジェクトのタスクを読み取り専用で現プロジェクトのガントに表示できるようにした。右クリックメニューの「🔗 参照を追加」またはツールバーの「🔗 参照」から追加でき、参照タスクはプロジェクト別のグループ行にまとまって表示されるため、日付・進捗をプロジェクト横断で確認できる。プロジェクトを跨いだ先行/後続タスクの設定にも対応（リンクのドラッグ操作、およびタスク詳細モーダルの「外部の先行タスク」チェックリストから設定可能）。参照タスク自体の編集は参照先プロジェクトで行う仕様とし、参照行から「参照先プロジェクトを開く」でジャンプできる。

### 変更
- **依存関係の循環を API 側で検証するようになった**: タスクの先行/後続関係（プロジェクトを跨ぐものを含む）に循環が生じる操作を API が `400 DEP_CYCLE_DETECTED` で拒否するようにした。従来は循環を含むデータの作成が可能だったため、API または MCP を直接利用しているクライアントでは、これまで許容されていたリクエストが拒否されるようになる破壊的変更となる可能性がある。

## [1.5.0] - 2026-07-05

### 追加
- **SQLite の定期バックアップ**: `api` プロセス内で起動時＋既定24時間ごとに DB のスナップショットを取得し、7世代を超えた古いバックアップを自動削除するようにした。間隔・保存先・世代数は環境変数（`BACKUP_INTERVAL_HOURS`/`BACKUP_DIR`/`BACKUP_RETENTION`）で調整でき、手動リストア手順も設計書に明記。
- **CI への E2E (Playwright) ジョブ追加**: GitHub Actions で `api`/`frontend`/`e2e` を起動して Playwright のE2Eテストを自動実行するようにした。失敗時はトレース・スクリーンショットをアーティファクトとして保存。

### 修正
- **E2E 設定のパス依存**: `playwright.config.ts` の `webServer` 起動コマンドがチェックアウト先の絶対パス（`/workspace/...`）をハードコードしており、CI 環境など配置場所が異なる環境では起動できなかった問題を、設定ファイルからの相対パス解決に修正。

## [1.4.1] - 2026-07-05

### 追加
- **CI への `npm audit` ゲート**: GitHub Actions に依存脆弱性チェックを追加（`api` は全依存、`frontend` は本番ビルドに同梱されない dev 専用依存を除外した `--omit=dev` で本番依存のみを対象）。

### 修正
- **`ws` の HIGH 脆弱性（メモリ枯渇 DoS）**: `ws` 8.0.0–8.20.1 に存在する tiny fragments/data chunks によるメモリ枯渇 DoS（GHSA-96hv-2xvq-fx4p）を 8.21.0 へ更新して解消。

## [1.4.0] - 2026-07-05

### 追加
- **エラー通知のトースト化**: `alert()` によるブロッキング通知と、無言で握りつぶしていた `.catch(() => {})` を、非モーダルのトースト通知（右下固定・種別ごとの自動消滅・手動クローズ）に置き換え、サイレント失敗を解消。プロジェクト一覧・タスク初回取得の失敗は、トーストに加えて再試行ボタン付きの可視エラー表示にした。
- **アクセシビリティの基本対応**: 記号・絵文字のみのアイコンボタン（展開/折りたたみ・開閉トグル・色パレット・フィルタクリア等）に `aria-label` を付与。
- **WebSocket のハードニング**: `maxPayload`（64KB）による過大メッセージの自動切断、および `Origin` ヘッダー検証（`CORS_ORIGIN` が `*` 以外の場合のみ）を追加。
- **CSV 式インジェクション対策**: CSV Export のユーザー入力由来列（`title`/`summary`/`description`/`assignee`）で先頭が `=+-@` のセルを無害化し、Import で往復復元できるようにした。
- **500系エラーメッセージのマスキング**: `statusCode>=500` のレスポンスは内部詳細を返さず汎用文言に置き換え、詳細はサーバーログにのみ出力するようにした。
- **本番 CORS 設定ガイド**: インターネット公開等で許可オリジンを絞り込む運用手順を `.env.example`/`docker-compose.yml` にコメントで明記。

### 変更
- **`GanttChart` コンポーネントの責務分割**: 約1560行あった `GanttChart.tsx` を、バー移動/リサイズ・依存リンクドラッグ・行D&D等の各フックと `WbsPanel`/`TaskContextMenus`/`GanttTimelineHeader`/`GanttSvgBody` 等のコンポーネントへ分割し、本体を673行のオーケストレーション層に整理（挙動不変）。
- **API のデッドコード整理**: `taskService` の未使用コードを整理。

### 修正
- **行D&D失敗時にトーストが出ない**: WBS 行のドラッグ&ドロップによる並び替えが失敗した際、ロールバックはされるがエラー通知が表示されない欠落を修正し、他の操作と同じくエラートーストを表示するようにした。
- **predecessor 削除時の CASCADE 検証整備**: タスク削除時の依存関係（predecessor）カスケード処理・CSV 中和とエスケープの組み合わせに関するテストを整備。
- **E2E テストの日付依存・旧ラベル参照**: 固定日付のためガントの既定表示範囲から外れて失敗していたテストを実行日からの相対日付に変更し、UI 変更（「＋ 子追加」フライアウト化）に追従できていなかったテストを更新。

## [1.3.2] - 2026-07-04

### 追加
- **CI（GitHub Actions）**: push/PR で `api`/`frontend` の typecheck・lint・test を自動実行するワークフローを追加。
- **ESLint（flat config）**: `api`/`frontend` に ESLint を導入（frontend は `eslint-plugin-react-hooks` を含む）。`typecheck`/`lint` の npm scripts を追加。
- **Docker ヘルスチェック**: `docker-compose` の `api` に `/health` を用いたヘルスチェックを追加し、`frontend` は `depends_on: service_healthy` で起動を待ち合わせるようにした。

### 修正
- **Import/Export でのタスク属性消失**: import/restore・CSV の往復（export→import→再export）およびタスク新規作成時に `titleColor`/`titleBgColor`/`estimateMinutes` が失われることがあった問題を修正。
- **並び替え（reorder）の循環・境界検証**: 並び替えで循環参照が発生しうる問題（一括指定の組み合わせを含む）と、プロジェクト境界を跨いだ並び替えを許容していた問題を修正し、検証を追加。
- **バッチ作成の検証強化**: タスクの一括作成（batch）で単一作成と同等のバリデーション（`status`/`priority`/`progress`・`parentId` 等）が効いていなかった問題を修正。
- **タスク一覧 `limit` の範囲検証**: `GET` タスク一覧の `limit` クエリが負値・巨大値・非整数を無検証で受け付けていた問題を修正し、範囲検証を追加。
- **設計書の誤記修正**: §2.1 の WebSocket プロセス構成に関する誤記（REST とは別プロセスと記載していた）を修正。

### 変更
- **フロントエンドの Vite 設定を一本化**: 重複していた `vite.config.js` を削除し、`vite.config.ts` に統一。

## [1.2.0] - 2026-06-20

### 追加
- **予定工数（タスクごとの工数設定）**: タスク編集に「予定工数」欄を追加。`3d`（3人日）/`4h`/`30m`/`1w`/`1d 4h`/`7:45` のような単位トークン・時:分で入力でき（`?` 書式ヘルプ付き）、内部は分で保持。Import/Export にも対応。
- **リソースビューを工数ベースの稼働率表示へ刷新**: 予定工数をタスク期間の稼働日へ均等配分した日次需要を、1日のキャパシティで割った**稼働率**を担当者×期間で表示。`〜80% 余裕／〜100% 適正／〜120% 注意／>120% 過負荷`のバンド着色で**過負荷を一目で把握**でき、各担当者の合計工数・ピーク稼働率を併記。セルにマウスオーバーすると、その%の**内訳**（寄与タスクごとの時間・合計需要・1日キャパ）を表示。
- **リソース設定（キャパシティ・稼働日）**: 1日のキャパシティ（既定 8:00、`7:45` 等の半端も可）と稼働日（既定 月〜金）を設定可能に。**アプリ既定**（ハンバーガーメニュー「リソース設定」）と**プロジェクト個別の上書き**（プロジェクトタブ右クリック「リソース設定」・「アプリ既定を継承」トグル）の2段。全ユーザー共有でサーバ保存。
- **リソースビューの高さ調整**: パネル上端の境界線をドラッグして高さを増減でき、担当者が多くてもガント本体を覆わない（高さは保持・超過分は縦スクロール）。

### 変更
- **リソースビュー見出し**: 「担当者別負荷」→「**担当者別 工数負荷**」に改称し、タスクの予定工数設定とのつながりを明確化。
- **Import/Export データ形式 1.1**: 予定工数 `estimateMinutes` を往復対象に追加（旧データは未設定として取り込む後方互換）。

### 修正
- **リソースビューの日付ズレ**: リソースビューの日付列がガント本体より1日前から表示されることがあった問題（UTC+9 等のタイムゾーンで発生）を、ガントと同じローカル日付グリッドに統一して修正。

## [1.1.1] - 2026-06-18

### 変更
- **マイルストーンを1点・不動に統一**: ガント上のマイルストーン（菱形）は移動・リサイズ不可とし、クリックでモーダルを開くのみに統一。WBS でもマイルストーン行の終了日を編集不可にし、開始日の編集時に終了日を同値へ追従させて常に1点を保つようにした。

### 修正
- **ヘッダーのマイルストーン強調範囲**: ヘッダーのマイルストーン強調（背景色）を日・曜日セルのみに限定。マイルストーンが週頭・月初・年初に重なると週/月/年セル全体まで色づいていた問題を修正。
- **バージョン表示の不整合**: `api`/`frontend` の `package.json` のバージョンが 1.0.1 のまま 1.1.0 にバンプされておらず、`/health`・ハンバーガー表示が実際のリリースと食い違っていた問題を解消（1.1.1 に統一）。

## [1.1.0] - 2026-06-17

### 修正
- **プロジェクト切替時の白画面**: 行仮想化で前プロジェクトのスクロール位置が残ると可視範囲が空になり画面が真っ白になることがあった問題を修正（スクロール位置のクランプ＋切替時の先頭リセット）。

## [1.0.0] - 2026-06-13

- TreeGantt 1.0 リリース。プロジェクト／タスクの CRUD、ツリー（WBS）＋ガントの統合ビュー、親サマリー・進捗集計、マイルストーン、依存関係（先行/後続）と矢印、クリティカルパス、リソースビュー、ドラッグ操作（移動／リサイズ／並べ替え／コピー＆挿入）、Markdown、CSV/JSON インポート・エクスポート、WebSocket によるリアルタイム同期、バージョン表示（ハンバーガー／`/health`）。
